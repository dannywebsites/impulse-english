# Agent 06.5 — Taste Quality Check

**Skill:** `taste` (Editorial-Minimal archetype)
**MCP:** `puppeteer` (DANGEROUS MODE)
**Trigger:** Runs per page, AFTER Visual QA Agent passes.
**Requires:** Visual QA report showing PASS, brand context report.

---

## Role

Enforce design quality and eliminate generic AI aesthetic patterns from page output.
This agent audits and flags — it does not fix.

PASS → proceeds to Quality Gate (Agent 07)
FAIL → returned to Optimisation Engine (Agent 04) for design fixes
CONDITIONAL PASS → documented fixes required, then re-check

## Configuration Dials — Blog Posts

Blog articles use the **Editorial-Minimal** archetype with these settings:

| Dial | Value | Rationale |
|------|-------|-----------|
| `DESIGN_VARIANCE` | **5** | Structured but not rigid — allows asymmetric elements within clean editorial flow |
| `MOTION_INTENSITY` | **3** | Minimal motion — readability first, no distracting animations on content pages |
| `VISUAL_DENSITY` | **3** | Generous whitespace — long-form reading comfort, breathing room between sections |

## Taste Quality Checklist

Record PASS / FAIL / FLAG for every item.

### Section 1 — Anti-Slop Visual Patterns

```
[ ] No equal-width columns forced on every layout element
[ ] No cards forced to identical heights with min-h hacks
[ ] No text-center applied to body paragraphs (only headlines and CTAs)
[ ] No generic shadow-md applied uniformly to all elements
[ ] No generic hero pattern (large heading + subtext + button + stock background)
[ ] Section padding varies appropriately (not uniform py-16 everywhere)
[ ] Visual hierarchy is clear — H1 > H2 > H3 sizing and weight are distinct
```

### Section 2 — Anti-Slop Content Patterns

```
[ ] No generic section headings: "How It Works", "Features", "About Us",
    "Our Process", "Why Choose Us", "What You Get"
    → Headings must communicate specific value when scanned alone
[ ] No superlatives: "Revolutionary", "Game-changing", "Supercharge",
    "Cutting-edge", "Seamlessly", "Unlock"
[ ] No vague social proof: "Trusted by 1,000+ companies" or round numbers
[ ] No generic testimonials: "Great product! — John D."
[ ] No placeholder text or Lorem ipsum
[ ] No emoji used as bullet points or list markers
[ ] Content matches Impulse brand voice (energetic, warm, authentic, expert)
```

### Section 3 — Typography & Spacing

```
[ ] Heading hierarchy has clear visual weight differentiation
    (H1 significantly larger/bolder than H2, H2 clearly distinct from H3)
[ ] Body text is left-aligned (not center-aligned)
[ ] Line height is comfortable for reading (1.5-1.75 for body text)
[ ] Paragraphs have adequate spacing between them
[ ] Blog content area has appropriate max-width for readability (600-800px)
[ ] Font sizes are appropriate for mobile reading (min 16px body)
```

### Section 4 — Layout & Structure

```
[ ] Sections have varied visual rhythm (not identical padding/structure repeated)
[ ] Images are properly sized and positioned within content flow
[ ] No orphaned elements (single line at bottom of a section, widowed headings)
[ ] FAQ sections use clean accordion or list — not card grid
[ ] CTA placement feels natural, not forced after every section
[ ] Mobile layout stacks cleanly without horizontal overflow
```

### Section 5 — Brand Consistency

```
[ ] Colors match Impulse palette: brand-red #ea4e59, accent-blue #12477d, zinc-surface #F4F4F5
[ ] No pure black (#000000) — uses slate-950 or zinc-950
[ ] No off-brand colors introduced
[ ] No external images — all from /assets/images/
[ ] Alt text is descriptive and in Peninsular Spanish
[ ] Overall visual impression matches Impulse brand: professional, warm, energetic
```

### Section 6 — Code Quality (if applicable)

```
[ ] No inline styles for positioning
[ ] No !important overuse (max 2 per file)
[ ] No z-index above 50
[ ] No hardcoded hex colors inline — uses Tailwind classes or CSS variables
[ ] Animations (if any) only use transform and opacity (GPU-safe)
```

## Gate Decision

### PASS
All sections green → Proceeds to Quality Gate (Agent 07)

### CONDITIONAL PASS
Minor flags only (cosmetic issues that don't affect readability or brand)
→ Document each flag with exact fix instruction
→ Return to Optimisation Engine (Agent 04) for fixes
→ Re-run Taste Check after fix

### FAIL
Multiple anti-slop patterns detected / brand violations / poor readability
→ Status: DESIGN QUALITY FAILED
→ Return to Optimisation Engine (Agent 04) — must address all flags before re-check
