<decisions>
## Implementation Decisions

### Astro page meta tags
- **D-01:** Each .astro page imports `NAP` from `../../utils/napData` (or appropriate relative path) and uses template literals for meta title, description, and keywords. Pattern: `title={`${pageSpecificPart} | ${NAP.name}`}`.
- **D-02:** Pages that already import NAP (from Phase 5/6 work) just need their hardcoded meta strings replaced. Pages that don't import NAP yet need the import added.
- **D-03:** The `description` and `keywords` props use NAP.name, NAP.city, NAP.neighborhood where "Impulse English Academy", "Madrid", "Barrio del Pilar" appear.

### Article data files (TypeScript)
- **D-04:** `data/articles/cambridge-b2-first.ts` and `cambridge-c1-advanced.ts` import NAP and replace all literal "Impulse English Academy" with `NAP.name` in their content strings (seoTitle, seoDescription, contextSections content, FAQ answers).
- **D-05:** Since these are TypeScript files with template literals, NAP.name can be interpolated directly: `` `En ${NAP.name}, te preparamos...` ``

### Article markdown files
- **D-06:** The 23 markdown files in `src/content/articles/` contain "Impulse English Academy" in frontmatter `paaAnswer`, `contextSections.content`, `brandSection.content`, and `faqItems.answer` fields. These are YAML strings, not TypeScript — they cannot import NAP.
- **D-07:** Strategy for markdown: The `brandSection` content already uses brand-config template system (from Phase 6). For other fields (paaAnswer, contextSections, faqItems), replace literal "Impulse English Academy" with a placeholder like `{{brandName}}` that the auto-publish pipeline resolves, OR accept that existing articles keep the current brand name and only NEW articles (generated by auto-publish) use dynamic values.
- **D-08:** Recommended approach: Replace "Impulse English Academy" in markdown with NAP.name value at build time using an Astro content collection transform, OR simpler — do a find-and-replace in markdown to use the brand name from brand-config and have the onboarding CLI do the same replacement. Since markdown can't import TS, the CLI must update these files during onboarding.

### Onboarding CLI updates
- **D-09:** The onboarding CLI adds a step to find-and-replace "Impulse English Academy" in all markdown article files with the new client's brand name.
- **D-10:** The onboarding CLI adds a step to update .astro page meta tags if any still contain hardcoded brand references (belt-and-suspenders approach alongside the NAP import changes).

### Verification approach
- **D-11:** Success criteria #3 (zero "Impulse" in dist/) means: after running onboarding with test data, `grep -r "Impulse" dist/` should return zero results. The source files will have NAP.name which evaluates to the test client name at build time.

### Claude's Discretion
- Exact page-specific meta title patterns (what comes before `| ${NAP.name}`)
- Whether to use a content collection transform vs CLI find-and-replace for markdown
- Grouping of .astro pages into batches for the plan
- Whether seo-system pages also need meta tag debranding

</decisions>
